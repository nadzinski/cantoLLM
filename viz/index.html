One real speculative run: draft + main, greedy. The draft proposes up to 10 tokens; the main model verifies them all in one forward, keeps the matching prefix, fixes the first mismatch (or banks a bonus token), and both KV caches roll back the rest. The final output is token-for-token identical to running the main model alone — that's the correctness property.
